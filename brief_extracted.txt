Section 2 — Visual Direction and Tone
This site must position AlignAI as an enterprise-grade governance architecture — not a startup product, not a generic AI tool, and not a consulting brochure. Authoritative, structured, and restrained. Reference points: IBM.com, McKinsey Digital, Palantir.
✓  DO
✗  DO NOT
REFERENCE
Dark navy hero sections with structured layout
Gradient backgrounds or colourful hero images
IBM.com, Palantir.com
Flat matte colour panels throughout
Glassmorphism, neumorphism, or card drop shadows
McKinsey Digital
12-column grid, clear vertical alignment
Free-form asymmetric decorative layouts
Deloitte Insights
Minimal line icons — consistent stroke weight
Decorative illustration or startup blob art
Stripe.com icon style
White space used intentionally to signal authority
Dense content packing or visual busyness
Anthropic.com
Subtle CSS-based scroll reveals only
Parallax, animated particles, excessive motion
Linear.app motion
Enterprise photography if used: boardrooms, data
Stock AI imagery: glowing brains, circuit boards
Avoid photography when in doubt
Layout Principles
Full-width dark navy hero on every page. AlignAI logo top left. Navigation top right.
Content width: max 1200px centered. Generous horizontal padding on all sections.
Alternating section backgrounds: Navy → Off-White → White → Off-White. No all-white pages.
Mid Blue horizontal rule between every major section — 1px, full content width.
CTA buttons: flat Mid Blue #407BB7 fill, white text, 4px border-radius. No pill buttons, no outlined buttons.
No sidebars. Single-column content flow on all pages.
Mobile responsive. Breakpoints at 768px (tablet) and 390px (mobile).

Section 7 — Global Floating Chatbot Widget
7.1  Phase 1 — Visual Stub (Milestone 1)
Fixed position: bottom-right corner, 28px from bottom and right edges. Present on all 7 pages.
Trigger: 52px diameter circle, Mid Blue #407BB7, white chat SVG icon. Scale 1.06 on hover (200ms ease).
Click opens chat panel above the button — ease-in-out slide animation, 200ms.
Panel: 320px wide, Navy #0C1E39 background, rgba(99,188,231,0.2) border, 8px border-radius.
Header: Deep Blue #274185, shows AlignAI wordmark + 'Ask about AlignAI' label + 'Beta' badge.
Opening message: 'Hi — I can answer questions about the AlignAI framework, the AI Decision Visibility Assessment, or enterprise AI governance more broadly. What would you like to know?'
Input field and send button are visible in Phase 1 but non-functional (no API calls).
Footer inside panel: 'Powered by AlignAI  ·  Phase 2 — Coming Soon'
7.2  Phase 2 — Live Widget (Milestone 3)
API: Anthropic Messages API at /v1/messages. Model: claude-sonnet-4-20250514 or current Sonnet.
System prompt: AlignAI canonical context document (supplied by client before M3 begins).
Scope: answers both AlignAI framework questions and broader AI governance questions.
All API calls routed through Netlify Function or Vercel Edge Function (serverless proxy).
API key is never in client-side JavaScript or any browser-accessible file — see security note below.
Conversation history maintained within the current session only. No cross-session persistence.
Basic rate limiting on the serverless proxy function.
SECURITY — NON-NEGOTIABLE
The Anthropic API key must NEVER appear in any client-side code, HTML, JavaScript file, or public repository file. All API calls must route through a serverless proxy that holds the key as an environment variable. Any implementation that exposes the API key in front-end code will fail the M3 acceptance review and must be rebuilt before payment is released.
Section 8 — Technical Requirements
8.1  Technology Stack
Static site — plain HTML/CSS/JS or Astro/Next.js static export. No WordPress, Webflow, or page builders.
Hosting: Netlify or GitHub Pages. Domain: bytestream.ca.
Google Fonts: Sora (weights 300–800) and Inter (weights 300–600). Preloaded in <head>.
Post listing on Page 6 powered by /data/posts.json — no CMS required at launch.
Chatbot Phase 2: Netlify Function or Vercel Edge Function as API proxy.
Client area Phase 4: Auth0, Clerk, or Supabase Auth — confirm choice with client before M3 begins.
8.2  Performance
Lighthouse Performance score ≥ 90 on all pages, measured at 1440px desktop width.
All images: WebP format, lazy-loaded, explicit width and height attributes on every element.
No hero video backgrounds. Critical CSS inlined. Fonts preloaded in <head>.
8.3  Accessibility
WCAG 2.1 Level AA minimum compliance. Lighthouse Accessibility score = 100.
All colour combinations meet 4.5:1 contrast ratio minimum.
All interactive elements keyboard-navigable. Alt text on all non-decorative images.
8.4  SEO
Unique meta title and meta description on every page.
Open Graph tags on all pages for LinkedIn share previews.
JSON-LD structured data for Organisation on the Home page.
robots.txt and sitemap.xml included at launch.
8.5  Code Handoff
Source code in private GitHub repository — client invited as collaborator before M2 payment.
posts.json schema documented in README with step-by-step instructions for adding new posts.
Netlify / GitHub Pages deployment configuration documented in README.
All environment variable keys listed in README — values excluded from version control.
